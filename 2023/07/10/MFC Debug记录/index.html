# MFC Debug 记录

  • C++ 编译时间、PCH、stdafx.h

预编译头文件（.pch 文件）是把一个工程中较稳定的代码预先编译好放在一个中间形式的文件中，编译器可以更加快捷的使用这个文件，减少编译时间。预先编译好的代码可以是任何 C/C++ 代码。

预编译头文件的目的是提高编译速度。编译器以文件为单位编译，如果修改了一个工程中的一个文件，则所有文件都要重新编译，包括头文件里的所有东西。在较大的 C/C++ 程序中，这些头文件中所包括的东西往往很大，编译时间很长。其中一部分头文件不常被修改，为了减少因为修改 A 文件而去编译没有任何修改的 B 文件，将 B 文件加入 PCH 文件中，避免重复编译（比如 TinyXML）

一般情况下，预编译头文件使用 stdafx.h 和 stdafx.cpp；一可以用其他文件名

预编译头文件应当只 include 大的并且影响编译速度的头文件，例如 windows.h、STL、TiXML 等
